## 🔴 What is Broken / Not Working

### 1. **BADGES / ACHIEVEMENTS**
- **Status:** ❌ HARDCODED MOCK DATA
- **Root Cause:** The `achievements` array in `StudentDashboard.js` (lines 34–41) is **static JavaScript code**, not loaded from any API.
- **Example:** "Aptitude Ace", "Project Pro", etc. — same for every student, every login.
- **No backend endpoint** for badges/achievements exists in `server.py`.
- **Fix Needed:** Create `badges` collection in MongoDB, create `GET /api/badges` endpoint, create earning logic (e.g., earn "Project Pro" when 1st project is added).

---

### 2. **BROWSE OPPORTUNITIES / INTERNSHIPS**
- **Status:** ❌ NO DATA
- **Root Cause:** The database has **0 internship records**. Students see an empty page because nobody (placement staff or employer) has posted any internships yet.
- **Frontend:** Correctly fetches from `GET /api/internships` — API works fine.
- **Fix Needed:** Seed the database with sample internships OR encourage placement staff to post. Add a "seed data" script.

---

### 3. **VIEW GLOBAL OPPORTUNITIES**
- **Status:** ❌ BROKEN LINK
- **Root Cause:** The button in `StudentDashboard.js` (line 381) opens `https://internmatch.example.com` — this is a **placeholder/fake URL** that doesn't exist.
- **Fix Needed:** Replace with a real external job portal link (e.g., LinkedIn, Internshala, Unstop) OR build a real global opportunities section.

---

### 4. **UPDATING PROFILE (personal info like name, university, major)**
- **Status:** 🟡 PARTIALLY BROKEN
- **Root Cause:**
  - The frontend `profile` state is loaded from `GET /api/students/profile` which only returns: `skills, interests, resume_url, cover_letter, cgpa, graduation_year`.
  - Fields like `name`, `email`, `phone`, `university`, `major`, `linkedin`, `bio` are shown in the Profile form but **these are not in the `StudentProfile` model** (`server.py` lines 68–76).
  - When "Save Profile" is clicked, only the fields in `StudentProfile` are sent. Name, university, major, phone, linkedin, bio are **silently ignored and never saved**.
- **Fix Needed:** Expand `StudentProfile` model to include `name, phone, university, major, linkedin, bio` fields. Also update the `users` collection when name/phone changes.

---

### 5. **UPDATING RESUME**
- **Status:** 🟡 PARTIALLY WORKING
- **Root Cause:**
  - `resume_url` field IS in the `StudentProfile` model and IS saved to DB when "Save Profile" is clicked.
  - However, there's no file upload — only a URL text field. Students must manually paste a Google Drive or similar link.
  - The "Download CV" button does nothing (no `onClick` handler, line 605–608).
  - The Cover Letter template selector (Professional/Creative/Academic) does nothing — no real functionality.
- **Fix Needed:** Add file upload capability OR improve URL-based resume with a working download. Fix the "Download CV" button.

---

### 6. **UPDATING SKILLS**
- **Status:** ✅ WORKS (but lost on refresh if profile not saved)
- **Root Cause:** `addSkill()` / `removeSkill()` update local React state. Skills are saved ONLY when "Save Profile" button is clicked. Skills vanish if user navigates away without saving.
- **Fix Needed:** Auto-save skills on add/remove OR make the save more prominent.

---

### 7. **CERTIFICATES (Viewing / Adding)**
- **Status:** ❌ EMPTY
- **Root Cause:** **0 certificates in database.** Certificates are only generated by Faculty via `POST /api/certificates/generate` after an application is `approved`. Since there are no internships → no applications → no approvals → no certificates.
- **Add Certificate button** on the frontend has **no `onClick` handler** — it's a non-functional button (line 648–651).
- **Fix Needed:** Add manual certificate upload for students (with title, platform, date). Fix "Add Certificate" button.

---

### 8. **PROJECT SHOWCASE – Adding Projects**
- **Status:** ❌ BROKEN
- **Root Cause:**
  - The Project Showcase section shows **2 hardcoded projects**: "E-commerce Website" and "Mobile Weather App" — same for every single student (lines 694–727).
  - The "Add Project" button has **no `onClick` handler** — clicks do nothing.
  - **No backend API** exists in `server.py` for projects (`/api/projects` does not exist).
- **Fix Needed:** Create `projects` collection in MongoDB. Create `GET/POST/DELETE /api/projects` endpoints. Build an "Add Project" modal form. Remove hardcoded projects.

---

### 9. **LEARNING – Challenges & Skill Zones**
- **Status:** 🟡 HARDCODED / NON-FUNCTIONAL
- **Root Cause:** All learning content is static:
  - `learningResources` (lines 59–63) — 3 hardcoded items (Coursera, edX, NPTEL)
  - `challenges` (lines 66–70) — 3 hardcoded challenges
  - "Enroll Now" and "Start" buttons have **no `onClick` handlers** — do nothing
  - Soft-Skill Zone buttons (Mock Interview, Email Etiquette, etc.) have **no `onClick` handlers**
  - Aptitude Practice buttons (Logical Reasoning, Quantitative) have **no `onClick` handlers**
  - Progress Tracker shows **hardcoded numbers**: "3/8 courses, 12/20 challenges, 85% aptitude"
- **Fix Needed:** Create backend APIs for learning content. Add quiz engine. OR integrate with external platform APIs (NPTEL, Coursera). Fix all buttons to link to real content.

---

### 10. **QUIZZES**
- **Status:** ❌ COMPLETELY MISSING
- **Root Cause:** No quiz component exists anywhere in the codebase. The aptitude practice buttons show UI but nothing happens.
- **Fix Needed:** Build a quiz page/component with a question bank.

---

### 11. **NOTIFICATIONS (Real-time)**
- **Status:** ❌ NO NOTIFICATIONS EXIST
- **Root Cause:** 0 notifications in DB for this user. Notifications are only created as side effects of other actions (internship match, application status change, certificate generation, interview scheduled). Since no internships/applications exist, no notifications are ever created.
- **Fix Needed:** Seed some sample notifications OR ensure the workflow creates them (post internships → notifications fire automatically).

---

### 12. **SAME DATA FOR ALL USERS**
- **Status:** ❌ CONFIRMED BUG
- **Root Cause (Multiple):**
  - Achievements: Same 6 hardcoded badges for everyone
  - Skill Analytics radar chart: Same hardcoded values (90, 75, 80, 85) for everyone
  - Learning Resources: Same 3 hardcoded courses for everyone  
  - Challenges: Same 3 hardcoded challenges for everyone
  - Project Showcase: Same 2 hardcoded projects for everyone
  - Company Profiles: Same hardcoded Google/Microsoft entries for everyone
  - Progress Tracker: Same "3/8, 12/20, 85%" values for everyone
- **Fix Needed:** All of the above need to be fetched from per-user database records.

---

### 13. **SIGNUP – No Initial Data Collection**
- **Status:** 🟡 INCOMPLETE
- **Root Cause:** Registration (`Register.js`) only collects: `name, email, password, role, department, phone, organization`. Student profile is created as completely blank.
- The user expects students to fill in their data (university, CGPA, major, skills, graduation year) at signup.
- **Fix Needed:** Add an "onboarding" wizard after first login that guides students to complete their profile (university, major, CGPA, graduation year, skills, bio).
